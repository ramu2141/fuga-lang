# Fuga言語 処理系 (f-lang)

特異なメモリ消滅規則を持つ静的型付け言語「Fuga言語」の処理系（インタプリタおよびコンパイラ）である。

## 概要

本プロジェクトは、Fuga言語のインタプリタおよびコンパイラの開発を行うものである。
Fuga言語は、「一度参照された変数は消滅する」という厳密なライフサイクルルールを持つ言語である。現在、この言語仕様に基づく抽象構文木（AST）の構築、動的に評価・シミュレートする**インタプリタ機能**、およびC言語トランスパイルを経由したネイティブバイナリ生成を行う**コンパイラ機能**の両方が実装されている。

## 言語の特徴と処理系の注意事項

本処理系は、[fuga-spec.md](fuga-spec.md) に準拠して以下の言語仕様を厳密に処理する。

1. **変数の消滅規則**: 
   変数は評価（参照）された直後に消滅する。消滅した変数にアクセスすると、インタプリタ・コンパイラ（実行時）ともに即座に未定義参照エラーとして異常終了する。
2. **自己代入による再定義**: 
   `$x = $x + 1.` のように、評価によって消滅した変数を再度同じ変数に代入（自己代入）することで、変数を再定義・延命することができる。
3. **ダイナミックスコープと読み取り専用規則**: 
   関数呼び出しにおいて、変数はレキシカルスコープではなく実行時のコールスタックを親方向に遡って探索される。呼び出し元から探索された外部スコープの変数は「読み取り専用」として扱われ、評価しても消滅しない。
4. **例外処理の制御フロー**: 
   `loop` の脱出などには `throw` と `try-catch` 構文を利用する（コンパイラでは `setjmp/longjmp` を用いて例外を巻き戻す）。

## ビルド・テスト方法

本プロジェクトはRustで実装されている。以下のコマンドでビルドおよびテストが可能である。

### ビルド

```bash
# デバッグビルド
cargo build

# リリースビルド
cargo build --release
```

### 処理系の実行 (インタプリタ / コンパイラ)

開発環境では `cargo run` コマンドから直接実行できます。`-c` または `--compile` フラグの有無によってインタプリタ・コンパイラの挙動を切り替えます。

また、事前に `cargo build --release` でビルドした実行可能ファイル (`./target/release/f-lang`) を直接使用することも推奨されます。

```bash
# インタプリタモード（ソースコードを直接評価・実行する）
cargo run -- <fugaソースファイル>
# または
./target/release/f-lang <fugaソースファイル>

# コンパイラモード（ソースコードからネイティブバイナリを生成する）
# ※内部で gcc を呼び出します
cargo run -- <fugaソースファイル> -c
# または
./target/release/f-lang <fugaソースファイル> -c

# 出力ファイル名を指定してコンパイルする
cargo run -- <fugaソースファイル> -c -o <出力パス>

# バイナリ生成と同時に、トランスパイルされた C言語のソースコードも出力する
cargo run -- <fugaソースファイル> -c --emit-c
```
※正常に終了した場合は、Fuga言語の仕様に従い終了コード `0` を返す。

### テストの実行

16個のテストスクリプトを用いた統合テスト（変数消滅、スコープ、ループ制御、算術演算などの網羅的検証）が用意されている。コンパイラ・インタプリタ双方での挙動がテストされる。

```bash
# 統合テストの実行
cargo test --test integration_test
```
