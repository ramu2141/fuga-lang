# fuga-helper スキル

本スキルは、fuga言語のソースコードの記述、およびコンパイラ・インタプリタ開発時に、コードが正式仕様（`fuga-spec.md`）に完全に準拠しているかを自動・手動でセルフチェックするためのガイドラインである。

---

## 1. fuga言語コード記述時のチェックリスト

fuga言語のソースコード（例: テストコードなど）を生成または修正する際、以下の項目を必ずチェックすること。

### ① 全ての文の終端記号（ピリオド）
*   **規則:** すべての文（変数宣言、式、関数定義、loop、try-catchを含む）の最末尾にピリオド（`.`）が付与されているか。
*   **NG例:**
    ```
    func i32 #foo() = {
        $foo = 1.
    } (ピリオドがない)
    ```
*   **OK例:**
    ```
    func i32 #foo() = {
        $foo = 1.
    }. (末尾にピリオドが必要)
    ```

### ② 定数の16進数大文字表記
*   **規則:** 数値リテラルに `0x` などのプレフィックスが付いていないか。英大文字（`A`-`F`）のみが使用されているか（小文字は禁止）。
*   **NG例:** `0x1a.`, `1f.`
*   **OK例:** `1A.`, `1F.`

### ③ 変数の消滅違反 (Destruction Violation)
*   **規則:** 同一の式（左から右への評価）の中で、同じ変数が2回以上評価されていないか。
*   **NG例:**
    ```
    var i32 $y = $x + $x. (2つ目の $x を評価する時点で、最初の $x は消滅しているためエラー)
    ```
*   **OK例 (複製して使用):**
    ```
    (グローバル経由や補助関数で $x_copy を作ってから演算する)
    var i32 $y = $x + $x_copy.
    ```

### ④ 自己代入の適用条件
*   **規則:** 自己代入（自動再定義）を期待する式において、右辺に左辺の変数が**「正確に1つだけ」**含まれているか。
*   **NG例:** `$x = $x + $x.` (右辺に2つ含まれるため自己代入は適用されず、右辺評価中に消滅エラー)
*   **OK例:** `$x = $x + 1.` (右辺に1つだけなので、代入完了時に `$x` は有効なまま更新される)

### ⑤ 外部スコープのリードオンリー参照（コピーハック）
*   **規則:** 変数を消滅させずにコピーする際、ダイナミックスコープを利用して「現在実行中の関数スコープ外」から参照する構造になっているか。
*   **イディオム:**
    ```
    var i32 $a = A.
    func i32 #copy_a = { $copy_a = $a }. (外部の $a を参照するため $a は消滅しない)
    var i32 $a_copy = #copy_a.
    ```

### ⑥ 例外によるループ脱出
*   **規則:** `loop` 文から脱出するための `throw` と、それを受け止める `try-catch` が正しくネストされているか。
*   **イディオム:**
    ```
    try {
        loop {
            throw 1 @ $condition. (条件成立時に例外 1 を送出)
        }
    } catch 1 {
        (ループを抜けたあとの処理)
    }.
    ```

## 2. 処理系（コンパイラ・インタプリタ）実装時のチェックリスト

fuga言語のコンパイラやインタプリタのパーサ・アナライザを実装する際、以下の仕様をテストコードで網羅しているか確認すること。

1.  **変数宣言の初期値省略:** `var i32 $x.` がエラーにならず、`$x` の値が `0` で初期化されること。
2.  **配列要素ごとの消滅:** 配列 `$arr[1]` を評価したとき、`$arr[1]` のみが消滅し、`$arr[0]` や `$arr[2]` は有効なまま残ること。消滅した要素へのアクセスは、代入も含めてエラーになること。
3.  **引数の消滅:** 関数 `func void #foo(i32 $val)` に対し、`#foo($x)` と呼び出した場合、呼び出し元フレームで `$x` が即座に消滅すること。
4.  **暗黙の戻り値変数:** 関数 `#bar` の内部で `$bar` という変数が自動定義され、初期値 `0` であり、関数終了時にその値が返却されること。
5.  **組み込み関数のシグネチャと例外番号:**
    *   `#_putc(u8)`: 戻り値 `0`。エラー時例外 `FFFF0000`。
    *   `#_getc`: 戻り値 `u8`。エラー時例外 `FFFF0001`。
    *   `#_peek(u32)`: 戻り値 `u8`。エラー時例外 `FFFF0002`。
    *   `#_poke(u32, u8)`: 戻り値 `0`。エラー時例外 `FFFF0003`。
