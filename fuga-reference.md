# fuga言語 ユーザーリファレンス

fuga言語へようこそ！
fuga言語は、「一度使った変数は消滅する」という極めて過酷なメモリ管理仕様を持つプログラミング言語です。一見するとまともにプログラムを書くことが不可能なように思えますが、言語の持ついくつかの仕組み（ダイナミックスコープや自己代入など）をパズルのように組み合わせることで、高度な処理やループ処理を記述することができます。

本書では、fuga言語の基本的な使い方から、実用的なコーディングパターンまでをサンプルコードを交えて分かりやすく解説します。

---

## 1. クイックスタート (Hello World)

まずは、標準出力に "Hello, World!\n" と出力するプログラムを見てみましょう。

```
(グローバル配列に文字コードを格納)
array u8*D $hello = {48, 65, 6C, 6C, 6F, 2C, 20, 57, 6F, 72, 6C, 64, A}.

(変数のコピーを行う補助関数)
func i32 #copy_i = {
    $copy_i = $i. (呼び出し元の $i を参照。外部スコープなので消滅しない)
}.

(メイン処理を行う関数)
func i32 #main = {
    var i32 $i = 0.
    try {
        loop {
            (カウンタ $i を消滅させずに2つコピーする。ループ内で宣言した変数はループ終端で自動的に消滅する)
            var i32 $i_for_check = #copy_i.
            var i32 $i_for_array = #copy_i.

            (ループカウンタが配列長 D (13) に達したらループを抜ける。$i_for_check はここで消滅)
            throw 1 @ $i_for_check = D.

            (次のインデックスを計算。$i はここで消滅)
            var i32 $next_i = $i + 1.

            (コピーしたインデックスで配列を読み込む。$i_for_array はここで消滅)
            var u8 $char = $hello[$i_for_array].

            (文字を1文字出力。$char はここで消滅)
            var i32 $unused = #_putc($char).

            (次のループに向けて $i を再定義。$next_i はここで消滅)
            $i = $next_i.
        }.
    } catch 1 {
        (正常終了)
    }.
}.

var i32 $exit_code = #main.
```

## 2. 基本文法とデータの表現

### 2.1. 文の終わり
すべての文はピリオド（`.`）で終わります。ピリオドを忘れると構文エラーになります。

### 2.2. コメント
丸括弧 `( )` で囲まれた部分はコメントになり、実行時には無視されます。文の途中にも挿入可能です。
```
var i32 $a = 5. (ここはコメントです)
var i32 $b = (途中に挟む) 10.
```

### 2.3. 定数の書き方（16進数）
数値はすべて**16進数**で記述します。一般的なプログラミング言語のような `0x` や `h` といったプレフィックスは付けません。また、英字は大文字（`A`〜`F`）のみ使用可能です。
* `A` は 10進数の 10
* `10` は 10進数の 16
* `-1A` は 10進数の -26

## 3. 変数と「消滅」のルール

### 3.1. 変数宣言と初期値の省略
変数は、使用する前に型を指定して定義する必要があります。fuga言語では、変数宣言時に**初期値の指定を省略することが可能**です。初期値を省略した変数には、自動的に **`0`** が初期値として設定されます。

```
var i32 $count = 5. (初期値 5 で定義)
var u8 $flag.        (初期値を省略したため、自動的に 0 で初期化される)
```

### 3.2. 消滅の基本ルール
fuga言語の最大の壁は、**「値を読み出した（評価した）変数はその瞬間に消滅（無効化）する」**というルールです。

```
var i32 $x = 5.
var i32 $y = $x. (この行で $x の値を読み出したため、$x は消滅します)
var i32 $z = $x. (エラー！ $x は既に無効です)
```

この仕様を乗り越え、変数を安全に扱うための3つのテクニックを紹介します。

### テクニック1：ダイナミックスコープを利用した「値のコピー」
fuga言語は**ダイナミックスコープ**（関数呼び出し元を遡って変数を探す仕様）を採用しています。さらに、**「関数外のスコープの変数を参照した場合は、リードオンリーとなり、参照しても消滅しない」**という特別なルールがあります。

これを利用して、変数の値をコピーする関数を作成できます。

```
(呼び出し元の変数 $a を消滅させずにコピーする関数)
func i32 #copy_a = {
    $copy_a = $a. (外部スコープの $a を参照しているため消滅しない)
}.

func i32 #example = {
    var i32 $a = A.
    var i32 $b = #copy_a. (これで $b に A が代入される。$a は消滅していない！)
    var i32 $c = $a.       (ここで $a は消費され、消滅する)
}.
```

### テクニック2：自己代入による「変数の更新」
代入式の右辺に、左辺と全く同じ変数を1つだけ含む場合（自己代入）、例外的に**変数は消滅せずに新しい値で再定義**されます。

```
var i32 $count = 0.
$count = $count + 1. (有効: $count は消滅せず、1 に更新される)
```
※ただし、右辺に同じ変数が複数含まれる式（例: `$x = $x + $x.`）は、式の評価中に変数が消滅してしまうためエラーになります。

### テクニック3：`var` による「変数の再定義」
一度消滅してしまった変数を同じ名前で再度使いたい場合は、単に代入するのではなく、再度 `var` をつけて定義し直す必要があります。

```
var i32 $x = 1.
var i32 $y = $x. (ここで $x は消滅)
var i32 $x = 2.  (var を使った再定義なので有効)
```

### 3.4. 暗黙の型変換とコンパイル警告
fuga言語には型を強制的に変換するキャスト構文（`as` や括弧によるキャストなど）は存在しません。異なるデータ型同士で代入や引数の受け渡しを行うと、すべて**暗黙的（自動的）**に型が変換されます。

ただし、変換によって値が変わってしまったり、符号の解釈が狂う可能性がある場合は、コンパイル時に**警告（Warning）**が出力されます。

#### 1. 警告が出ないケース（安全な拡大）
元のデータ範囲が、代入先のデータ範囲に完全に収まる場合は安全に変換され、警告は出ません。
```
var u8 $a = 5.
var u16 $b = $a. (警告なし: 8ビット符号なし から 16ビット符号なし への自動拡大)
```

#### 2. 警告が出るケース（符号不一致・縮小変換）
*   **符号の不整合:** 符号ありと符号なしの型同士で変換する場合、ビット表現はそのままに値が再解釈されます。
    ```
    var i8 $c = -1. (16進数で FF)
    var u8 $d = $c. (警告あり: 符号の不整合。$d には 255 として格納される)
    ```
*   **ビット幅の縮小:** 大きい型から小さい型へ変換する場合、溢れた上位ビットは切り捨てられます（Trancation）。
    ```
    var u16 $e = 101. (10進数で 257。ビット表現は 0101)
    var u8 $f = $e.   (警告あり: 縮小変換。上位ビットが切り捨てられ $f には 1 が入る)
    ```

## 4. 配列の使い方

配列は、同じ型のデータを並べて管理する仕組みです。

### 4.1. 配列の定義
```
array i32*3 $arr = {1, 2, 3}. (要素数3の配列を初期値付きで定義)
```

要素数に変数を指定することもできますが、その場合は変数が消滅し、初期値の指定はできません（全要素は 0 で初期化されます）。
```
var i32 $size = 5.
array i32*$size $my_array. (ここで $size は消滅する)
```

### 4.2. 要素の消滅と再代入不可ルール
配列の要素（例: `$arr[1]`）を読み出すと、**その要素のみが消滅**します。他の要素は影響を受けません。
また、**一度消滅した要素スロットへの再代入（値の書き込み）はできません。**

```
array i32*3 $arr = {A, B, C}.
var i32 $val = $arr[1]. (ここで $arr[1] は消滅)
$arr[1] = 5.            (エラー！ 消滅したスロットへの再代入は不可)
var i32 $val2 = $arr[2]. (有効！ $arr[2] は消滅していない)
```

### 4.3. 外部配列のリードオンリー参照
関数内で、関数の外側（グローバルなど）で定義された配列の要素を参照した場合、**要素は消滅しません。**

```
array i32*3 $global_arr = {1, 2, 3}.

func i32 #read_global = {
    var i32 $val = $global_arr[0]. (外部の配列なので、$global_arr[0] は消滅しない！)
}.
```

## 5. 演算子と計算

fuga言語では、条件式を除き、**すべての演算は必ず代入を伴わなければなりません。** 単独の計算式（例: `$a + 1.`）はエラーになります。

また、**乗算 `*` や除算 `/` は言語仕様としてサポートされていません。** 積や商を求めるには、ビットシフトや加算・減算ループを組み合わせた補助関数（ユーザー定義関数）を実装する必要があります。

### 5.1. 演算子一覧（優先順位順）

fuga言語で利用可能な演算子は以下の通りです。表の上にあるものほど優先順位が高く、先に評価されます。

| 優先度 | 演算子 | 分類 | 結合規則 | 説明 | 例 |
| :---: | :---: | :--- | :---: | :--- | :--- |
| 1 | `~` | 後置単項 | なし | 指定された変数のビット反転値を返します。対象変数は消滅します。 | `$a = $b~.` |
| 1 | `?` | 後置単項 | なし | 変数の存在確認。有効（未消滅）なら 1、無効（消滅済）なら 0 を返します。**対象変数は消滅しません。** | `$a = $b?.` |
| 2 | `<<`<br>`>>` | 二項 | 左結合 | ビット左シフトおよび右シフト。Signed型（符号あり）は算術シフト、Unsigned型（符号なし）は論理シフトを行います。 | `$a = $b << 2.` |
| 3 | `&` | 二項 | 左結合 | ビット論理積 (AND)。 | `$a = $b & 2.` |
| 4 | `\|` | 二項 | 左結合 | ビット論理和 (OR)。 | `$a = $b \| 2.` |
| 5 | `+`<br>`-` | 二項 | 左結合 | 算術加算および減算。 | `$a = $b + 1.` |
| 6 | `<`<br>`>`<br>`=` | 二項(比較) | なし | 比較演算（小なり、大なり、一致）。真なら 1、偽なら 0 を返します。結合規則がないため、連続して記述できません。 | `$a = $b > 3.` |
| 7 | `=` | 代入 | 右結合 | 右辺の評価結果を左辺の変数に格納します。 | `$a = 1.` |

### 5.2. 一致比較と代入の識別

fuga言語では、一致比較も代入も同じ `=` 記号を使用します。一つの式に複数の `=` が記述されている場合、**最も左側にある `=` のみが代入演算子**として機能し、それ以外の `=` はすべて比較演算子として処理されます。

```
var i32 $a = 0.
var i32 $b = 3.
$a = $b = 3.
($b が 3 と一致するか比較し、結果である「1」を $a に代入します。このとき $b は評価されたため消滅します)
```

## 6. 条件分岐と繰り返し

### 6.1. 条件分岐（throw を使用したパターン）
fuga言語には `if` 文が存在しません。条件分岐を行うには、`try-catch` と `throw` を組み合わせて例外処理として記述します。

```
var i32 $a = 5.
try {
    ( $a が 0 より大きければ例外 1 を投げる。$a は消費される )
    throw 1 @ $a > 0.
    ( ここは $a が 0 以下の場合のみ実行される )
    var i32 $result = 0.
} catch 1 {
    ( $a が 0 より大きかった場合の処理 )
    var i32 $result = 1.
}.
```

### 6.2. 繰り返し処理 (loop)
`loop` は無限ループです。ループを脱出するには、`try` の中で `loop` を実行し、終了条件に達したときに `throw` でループ外の `catch` へジャンプします。

```
func i32 #copy_i = {
    $copy_i = $i.
}.

func i32 #main = {
    var i32 $i = 0.
    try {
        loop {
            var i32 $i_check = #copy_i.
            throw 9 @ $i_check = 5. ( $i_check が 5 になったら例外 9 を投げてループを脱出 )
            var i32 $next = $i + 1. ( インクリメントした値を計算 )
            $i = $next.             ( インクリメントした値で $i を蘇生 )
        }.
    } catch 9 {
        ( ループ脱出後の処理 )
    }.
}.
```

## 7. 関数の作り方と呼び出し

### 7.1. 関数の定義と戻り値
関数名は `#` から始まります。戻り値は、関数名プレフィックスの `#` を `$` に変更した変数に代入することで設定され、関数終了時に自動的に返却されます。

```
func i32 #double(i32 $val) = {
    $double = $val << 1. (戻り値変数 $double に $val の2倍の値を設定)
}.
```

### 7.2. 引数引き渡しの注意点
関数に実引数として変数を渡すと、**呼び出し元のスコープでその変数は即座に消滅**します。

```
var i32 $my_num = 5.
var i32 $res = #double($my_num). (ここで $my_num は消滅！)
var i32 $err = $my_num.          (エラー！ $my_num は使用できません)
```

## 8. サンプルプログラム集

### 8.1. 数値の掛け算（加算によるシミュレーション）
乗算 `*` 演算子がないため、ループと加算を用いて掛け算（`$a` × `$b`）を行います。

```
(変数コピー用の補助関数)
func i32 #copy_a = { $copy_a = $a. }.
func i32 #copy_b = { $copy_b = $b. }.

func i32 #multiply(i32 $a, i32 $b) = {
    var i32 $result = 0.
    try {
        loop {
            ( $b をコピーして終了判定を行う )
            var i32 $b_check = #copy_b.
            throw 1 @ $b_check = 0.

            ( $a, $b をそれぞれコピー )
            var i32 $a_copy = #copy_a.
            var i32 $b_copy = #copy_b.

            var i32 $next_a = $a. (コピー元はここで消費)
            $result = $result + $a_copy. (結果に足す)
            var i32 $a = $next_a. (元の変数を復元)

            var i32 $next_b = $b - 1. (減算。元の $b はここで消滅)
            $b = $next_b. (元の変数を復元)
        }.
    } catch 1 {
        ( $b が 0 になったらここへ来る。このとき $a は生きているが $b は消滅している )
        var i32 $unused = $a. (残った $a を破棄)
    }.
    $multiply = $result.
}.

func i32 #main = {
    var i32 $x = 3.
    var i32 $y = 5.
    var i32 $ans = #multiply($x, $y). ( 3 * 5 = 15 )
}.
```

### 8.2. キーボード入力のオウム返し (Echo)
標準入力から文字を入力し、改行コード（`A`）が入力されるまでそのまま出力し続けます。

```
(変数のコピーを行う補助関数)
func u8 #copy_char = { $copy_char = $char. }.

func i32 #echo_line = {
    try {
        loop {
            ( 1文字入力。例外 FFFF0001 (EOF/エラー) で終了 )
            var u8 $char = #_getc.

            ( 文字コードをコピーして改行チェックに使用する )
            var u8 $char_copy = #copy_char.

            ( 改行コード A であれば例外 2 を投げてループ終了 )
            throw 2 @ $char_copy = A.

            ( 1文字出力 )
            var i32 $unused = #_putc($char).
        }.
    } catch FFFF0001 {
        ( EOFまたは入力エラー時の処理 )
    } catch 2 {
        ( 改行検知による終了。最後に改行を出力 )
        var i32 $unused = #_putc(A).
    }.
    $echo_line = 0.
}.
```
